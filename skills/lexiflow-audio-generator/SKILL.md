# LexiFlow 听写音频生成器

## 概述

LexiFlow 是一个专业的听写素材生成工具，可以将单词或短语列表转换为高质量的 TTS 音频文件。它特别适用于语言学习场景，支持自定义重复次数和间隔时长，并能智能识别中英文以匹配最优音色。

**关键词**: 听写, 单词朗读, TTS, 语音合成, 复读, 英语学习, 中文学习, 音频生成, ListenHub, Marswave

## 核心能力

### 输入处理
- 接受单词/短语列表（逗号、换行、空格分隔均可）
- 支持上传 `.txt` 或 `.md` 文件批量导入
- 自动清洗和去重处理

### 智能语音合成
- 集成 ListenHub (Marswave AI) TTS API
- **智能语种识别**：自动检测每段文本是中文还是英文
- **动态音色切换**：为中文内容使用中式音色（如克克-1），为英文内容使用英式音色（如克克-2）
- 确保每种语言都使用最地道的发音

### 音频后处理
- 使用 FFmpeg 进行音频拼接
- 支持自定义重复次数（1-5次）
- 支持自定义间隔时长（2-10秒）
- 输出格式：MP3 (128kbps, 44.1kHz)

## 工作流程

```
用户输入文本 → 解析单词列表 → 语种检测 → TTS 生成 → FFmpeg 拼接 → 输出 MP3
```

### 详细步骤

1. **解析输入**
   - 清洗文本，移除多余空白
   - 按分隔符拆分为单词列表
   - 去重并保留顺序

2. **批量语音生成**
   - 对每个单词检测语种（中文/英文）
   - 根据语种选择最优音色
   - 调用 ListenHub API 生成单个音频
   - 下载并缓存临时文件

3. **音频合成**
   - 创建静音间隔文件
   - 按顺序拼接：单词音频 + 静音 + 重复...
   - 使用 FFmpeg concat 滤镜合并
   - 输出最终 MP3 文件

4. **清理与交付**
   - 删除临时文件
   - 返回最终音频路径或下载链接

## 技术要求

### 环境依赖
- Python 3.8+
- FFmpeg（已安装并配置到系统 PATH）
- 有效的 ListenHub API 密钥

### 环境变量
```bash
LISTENHUB_API_KEY=lh_sk_xxxxxx  # 必需
LISTENHUB_BASE_URL=https://api.marswave.ai/openapi/v1  # 可选
```

### API 端点
- `POST /api/generate` - 生成音频
- `GET /api/speakers` - 获取可用音色列表
- `GET /api/audio/{filename}` - 下载生成的音频

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `text` | string | 必填 | 单词列表文本 |
| `speaker_id` | string | `voice-clone-692f06bf2855dbc00c29a9c6` | 音色 ID |
| `repeat_count` | int | 1 | 每个单词重复次数 (1-5) |
| `interval_seconds` | float | 3.0 | 单词间隔秒数 (2-10) |

## 输出规范

- **文件格式**: MP3
- **编码**: libmp3lame, 128kbps
- **采样率**: 44100Hz
- **声道**: 立体声
- **命名规则**: `YYYYMMDD-HHMMSS.mp3`
- **存储位置**: `./OutputAudios/`

## 限制与边界

### 本技能适用于
- ✅ 批量生成单词/短语朗读音频
- ✅ 创建带重复和间隔的听写训练素材
- ✅ 中英双语内容的智能配音

### 本技能不适用于
- ❌ 实时语音合成或流式播放
- ❌ 长文章或段落级别的朗读（建议拆分）
- ❌ 音频编辑、剪辑、混音等后期处理
- ❌ 语音识别或转写

## 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| API 密钥无效 | 返回错误提示，要求检查 `.env` 配置 |
| 单个单词生成失败 | 跳过该单词，继续处理其余，最终报告失败列表 |
| FFmpeg 未安装 | 返回错误提示，指导安装 |
| 网络超时 | 自动重试 3 次，间隔 2 秒 |

## 代码风格指南

使用本技能生成代码时：
- 使用 `async/await` 处理 API 调用
- 使用 `tempfile` 管理临时文件
- 使用 `subprocess` 调用 FFmpeg
- 遵循 PEP 8 规范
